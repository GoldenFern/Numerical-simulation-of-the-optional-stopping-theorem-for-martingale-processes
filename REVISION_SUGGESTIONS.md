# 项目修改建议

## 总体架构

1. 保持现有“每章一个模型、每章三张图”的结构，不改变图表主题；改进重点放在生成流程、绘图一致性和论文表述严谨性。
2. 将 `src/core/visualization.py` 作为全项目唯一绘图风格入口，统一字体、字号、网格、颜色、误差线和保存路径。
3. 图表生成后同时保存到 `output/figures/` 和 `paper/figures/`，避免论文使用的图与最新实验图不同步。
4. 随机实验中应明确区分“可复现的随机种子”“有限时间截断估计”和“理论极限”。论文正文不能把有限截断或 Monte Carlo 噪声解释成理论结论。

## 图表绘制

1. 保留现有 15 张图：Moran 模型、保险破产、赌徒破产、秘书问题、美式期权各 3 张。
2. 所有图统一使用学术化的低饱和配色：蓝色表示模拟主结果，红色表示理论/失效对照，绿色/橙色用于辅助曲线，灰色用于参考线。
3. 单图标题应短，主要解释放在论文图注中；坐标轴使用中文说明并保留数学符号。
4. 半对数和双对数图必须显示主/次网格，帮助读者判断指数衰减或幂律衰减。
5. 误差棒统一表示 95% 置信区间，并在图例或图注中说明。
6. 美式期权路径图应区分“提前行权”和“到期/未提前行权”，不能把所有路径末端都标为行权点。
7. 第四章尾部参考线应标为 $C t^{-1/2}$，因为本实验的单边障碍为 $b=10$，不应直接套用 $b=1$ 的常数。

## 论文撰写

1. 各章建议采用固定逻辑：背景介绍、理论分析、数值实验设计、结果与讨论。现有结构基本正确，需修正文内不严谨或和数据不一致的表述。
2. 第一章应更明确说明五个模型的递进关系：OST 直接成立、构造辅助鞅、条件失效、最优停时推广。
3. 第三章要说明 Monte Carlo 破产概率是有限时间估计，因有限模拟时窗会低估最终破产概率，尤其在初始资本较大时更明显。
4. 第四章需要重点修正截断停时：对每个固定 $N$，$\mathbb{E}[S_{\tau\wedge N}]=0$；但 $S_{\tau\wedge N}\to S_\tau=b$ 几乎处处，不能交换极限与期望，这正是一致可积性缺失的体现。
5. 第五章 Snell 包络的图文解释要匹配当前定义：$V_n$ 是在第 $n$ 位候选人是 record 时的条件价值，前段为继续等待价值，阈值之后与即时收益重合。
6. 第六章应把 LSM 算法写成“现金流与行权时刻”的倒推形式，强调 continuation value 是折现到当前时刻的未来现金流回归。
7. 结论章应以表格总结五个模型中 OST 的角色，并补充数值方法的经验教训。

## LaTeX 排版

1. 不在 `\section{}` 标题中重复写“第一章”“第七章”，交给 LaTeX 自动编号。
2. 在 `main.tex` 中补全模板需要的 `\Archive{}`，否则干净编译会因 `\@Archive` 未定义失败。
3. 显式设置 `\graphicspath{{figures/}{../output/figures/}}`，确保论文图表路径稳定。
4. 使用 `hyperref` 的中文友好配置，避免目录、书签和引用出现编码或边框问题。
5. 表格列宽应为两栏论文服务，结论章总表保留 `table*`，但列描述要简洁，避免过宽文本挤压。
